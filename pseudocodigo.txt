Inicio del programa en CMD.

Imprimir token con el código - print("TOKEN:", os.getenv("GITHUB_TOKEN")) en main.py para asegurar funcionamiento de token para consulta
en archivo main.

Se descarga el programa con sus carpetas. Posteriormente sobre cmd se ejectuta:

docker build -t axanet .

docker run -it -e GITHUB_TOKEN=%GITHUB_TOKEN% -v "%cd%\clientes:/app/clientes" axanet

Debe de mostrar el token y el menu.

Se importa OS para la ubicacion de archivos guardado, modificacion, consulta e eliminicacion sobre windows enlazado a GitHub.

Se importa token para acceder al token creado para realizar el evento de consulta.

Se importa requests para obtener la API de dispatches del proyecto, convertir la informacion a Jason y los headers.


Funcion Notificar Cliente.

Variable token para cachar el token de GitHub.
variable url para cachar la direccion el API de dispatches del proyecto.
Diccionario Headers y data para validar token, crear evento y cachar nombre consultado.
Requests post para subir detalles de consulta a GitHub.

Diccionario Clientes vacio al inicio con variable carpeta clientes referenciando diccionario clientes.

Logica para crear carpeta en caso de no existir	para ir agregando clientes.


Funcion Crear Cliente.

Agregar nombre de cliente con input y cachar ruta con nombre de cliente creado.

Logica de no crear cliente si ya existe.

Agregar servicio de cliente con input

Generar archivo y cacharlo bajo la variable archivo.

Abrir archivo nuevo y agregar nombre y servicio.

Cachar bajo diccionario clientes nombre de clientes que viene de archivo con nombre y ruta.

Imprimir que el cliente fue creado correctamente.


Funcion consultar Cliente

Cachar en variable con input nombre de cliente

En caso de existir mostrar con R la informacion del cliente seleccionado

Mediante funcion consultar consulta con parametro nombre mostrar que cliente se reviso en workflow.
